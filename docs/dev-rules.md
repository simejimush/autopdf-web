# 開発運用ルール（AutoPDF）

## 1. 作業前コミット（必須）

作業開始前に必ずコミットする

## 2. 1変更1コミット

1つの機能・修正ごとにコミットする
（UI変更とロジック変更を混ぜない）

## 3. 危険ファイルは全面置換禁止

以下は丸ごと置き換え禁止

- app/(app)/rules/page.tsx
- RunButton.tsx

## 4. 作業終了時コミット（必須）

その日の作業終了時は必ずコミット

## 5. デプロイ前チェック（必須）

以下を手動確認してからデプロイする

- ルール作成
- 編集
- 実行
- 削除
- 検索

## 6. 監視 / 通知機能の変更ルール

- 通知実装（Slack / メール）と大きなUI変更を同時に行わない
- Webhook / メール送信を追加した場合は、本番投入前に必ずテスト送信を確認する
- 通知文言だけでなく、`runs.error_code` と整合しているか確認する
- 通知失敗時に本処理が止まらないことを確認する
- 管理者通知を入れる変更では、通知条件（どの error_code で飛ぶか）を事前に明示する
